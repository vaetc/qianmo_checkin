# QMJ自动签到脚本

自动签到并完成每日威望任务的 GitHub Actions 脚本。

## 功能特性

- ✅ 每天自动签到
- 🎁 自动完成每日威望红包任务
- 📊 获取威望和积分信息
- 🔄 支持手动触发
- 📝 详细的运行日志

## 工作流程

1. **每日签到** - 自动完成论坛签到
2. **领取任务** - 自动申请并完成每日威望任务
3. **查看威望** - 显示当前威望和积分

## 使用方法

### 1. Fork 本仓库

点击右上角 Fork 按钮

### 2. 获取 Cookie

1. 登录 [QMJ](https://www.1000qm.vip/)
2. 按 F12 打开开发者工具
3. 切换到 Network（网络）标签
4. 刷新页面
5. 找到任意请求，查看 Request Headers
6. 复制完整的 Cookie 值

Cookie 格式示例：bKHu_2132_saltkey=xxx; bKHu_2132_auth=xxx; bKHu_2132_sid=xxx


### 3. 配置 GitHub Secrets

1. 进入你 Fork 的仓库
2. 点击 Settings（设置）
3. 左侧菜单选择 Secrets and variables → Actions
4. 点击 New repository secret
5. Name 填写：`QIANMO_COOKIE`
6. Value 粘贴你的 Cookie
7. 点击 Add secret

### 4. 启用 GitHub Actions

1. 进入仓库的 Actions 标签
2. 点击 "I understand my workflows, go ahead and enable them"
3. 选择 "阡陌居自动签到" 工作流
4. 点击 "Enable workflow"

### 5. 测试运行

1. 在 Actions 页面选择工作流
2. 点击 "Run workflow" 手动触发
3. 查看运行日志确认是否成功

## 运行时间

- 自动运行：每天北京时间 00:00
- 手动运行：随时可在 Actions 页面触发

## 注意事项

⚠️ **Cookie 安全**
- Cookie 包含登录凭证，请妥善保管
- 不要将 Cookie 提交到代码中
- 定期更新 Cookie（如果登录状态失效）

⚠️ **使用限制**
- 遵守论坛规则
- 不要频繁运行脚本
- 建议仅用于个人账号

⚠️ **任务说明**
- 脚本会自动识别并完成所有可用的每日任务
- 如果任务已完成，会跳过该任务
- 部分任务可能需要特定条件才能完成

## 故障排查

### 签到失败
- 检查 Cookie 是否过期
- 确认论坛是否可以正常访问
- 查看 Actions 运行日志

### 任务领取失败
- 确认任务是否已经完成
- 检查账号权限是否足够
- 查看详细日志了解具体错误

### Cookie 过期
重新获取 Cookie 并更新 GitHub Secrets
